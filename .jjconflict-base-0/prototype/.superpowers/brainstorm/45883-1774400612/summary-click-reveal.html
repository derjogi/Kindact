How this works under the hood

The AI pre-computes overlapping reference spans at multiple granularities when generating the summary. Stored as:

{
  "content": "...usage data is biased toward neighborhoods...",
  "references": [
    { "start": 72, "end": 82, "text": "usage data",
      "commentIds": ["c35", "c35a", "c31a"],
      "strength": "direct" },
    { "start": 86, "end": 92, "text": "biased",
      "commentIds": ["c35a", "c35"],
      "strength": "direct" },
    { "start": 72, "end": 127,
      "text": "usage data is biased toward...infrastructure",
      "commentIds": ["c35", "c35a", "c33b"],
      "strength": "direct" },
    { "start": 86, "end": 127,
      "text": "biased toward...infrastructure",
      "commentIds": ["c35a", "c33b"],
      "strength": "approximate" }
  ]
}

When the user clicks a word, the UI finds all spans containing that character position, merges the comment IDs, and groups them by strength ("direct" → Related, "approximate" → See also). The surrounding context (neighboring words in matching spans) gets the faint yellow highlight.
