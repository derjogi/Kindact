A — Inline Highlights with Hover Preview

Key phrases in the summary are highlighted (subtle underline + warm tint). Hovering shows a compact preview of the source comment(s). Clicking scrolls to the full comment thread below.

Pros

  • Most natural reading flow — references are contextual
  • Immediately clear which parts of the summary are traceable
  • Hover = quick check, click = deep dive

Cons

  • Highlighted text could feel cluttered in a long summary
  • AI needs to identify which phrases map to which comments

B — Side Margin Sources

Summary text is clean (no inline markup). Source comments appear in a right margin panel, visually aligned with the paragraph they support. Clicking a source card scrolls to the full comment.

Pros

  • Clean reading experience — summary text is uncluttered
  • Sources are always visible, no hover needed
  • Academic/Wikipedia feel — fits Kindact's ethos

Cons

  • Takes more horizontal space
  • Harder to connect which source maps to which sentence on mobile

Data model implication

Both approaches need the same underlying data: the AI summary stored with a references array mapping text spans (or paragraph IDs) to comment IDs. The AI generates this mapping when it produces the summary. Example:

{
  "content": "...usage data is biased toward existing infrastructure...",
  "references": [
    { "phrase": "usage data is biased", "commentIds": ["c35", "c35a"] },
    { "phrase": "$200k figure is unrealistic", "commentIds": ["c32", "c32b"] },
    { "phrase": "flex-post protection", "commentIds": ["c36", "c36b"] }
  ]
}
